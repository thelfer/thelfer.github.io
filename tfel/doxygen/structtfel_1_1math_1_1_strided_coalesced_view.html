Detailed Description

template<MappableMathObjectUsingCoalescedViewConcept MappedType, typename IndexingPolicyType = typename MappedType::indexing_policy>
struct tfel::math::StridedCoalescedView< MappedType, IndexingPolicyType >

view of an object from a strided memory area

Template Parameters
MappedTypetype of the object mapped to the memory area

This view is similar to CoalescedView but stores only a base pointer and a stride instead of an array of pointers. This is more efficient for SoA layouts in terms of register usage, particularly on GPU where registers are easily saturated.

Memory layout (for a stensor<2> with 4 components and stride S):

  • Component 0: base_ptr + 0 * S
  • Component 1: base_ptr + 1 * S
  • Component 2: base_ptr + 2 * S
  • Component 3: base_ptr + 3 * S
