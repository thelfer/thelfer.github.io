The first version of transform::exe performs the operation op(*i) for each iterator i in the range [first, first+N), and assigns the result of that operation to *o, where o is the corresponding output iterator. The return value is result + N.

Parameters
InputIteratoriterator to the element to be transformed.
OutputIteratoriterator to which result is placed.
UnaryFunctionfunction used to perform transformation.
Returns
OutputIterator iterator after the last element transformed.
Precondition
  • InputIterator must be a model of Input Iterator.
  • OutputIterator must be a model of Output Iterator.
  • UnaryFunction must be a model of Unary Function.
  • InputIterator's value type must be convertible to UnaryFunction's argument type.
  • UnaryFunction's result type must be convertible to a type in OutputIterator's set of value types.

The second version of transform is very similar, except that it uses a Binary Function instead of a Unary Function: it performs the operation op(*i1, *i2) for each iterator i1 in the range [first1, last1) and assigns the result to *o, where i2 is the corresponding iterator in the second input range and where o is the corresponding output iterator. That is, for each n such that 0 <= i < N, it performs the assignment *(result

  • i) = op(*(first1 + i), *(first2 + i). The return value is result + N.

Returns
OutputIterator iterator after the last element transformed.
Precondition

  • OutputIterator must be a model of Output Iterator.
  • BinaryFunction must be a model of Binary Function.
  • InputIterator1's and InputIterator2's value types must be convertible, respectively, to BinaryFunction's first and second argument types.
  • UnaryFunction's result type must be convertible to a type in OutputIterator's set of value types.
